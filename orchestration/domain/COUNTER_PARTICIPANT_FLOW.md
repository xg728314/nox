\# COUNTER PARTICIPANT FLOW (LOCKED)



\## 0. 목적



카운터 화면에서 미지정 아가씨를 빠르게 여러 명 추가하고,

각 아가씨를 \*\*가게 → 종목 → 담당실장\*\* 순서로 확정할 수 있어야 한다.



이름은 나중에 수정 가능해야 하며,

종목/실장/가게가 확정되면 화면에 즉시 반영되어야 한다.



\---



\## 1. 기본 흐름



\### 1.1 추가



\* `+ 아가씨` 클릭

\* 미지정 participant 1개 생성



\### 1.2 설정 시작



\* 미지정 participant 클릭

\* 바텀시트 오픈



\### 1.3 바텀시트 단계



순서는 반드시 아래와 같다.



1\. `store` 선택

2\. `category` 선택

3\. `manager` 선택



\### 1.4 이름



\* 이름은 위 3단계 완료 후에도 비어 있어도 된다

\* 이름은 participant 카드의 네모 입력칸에서 나중에 수정 가능하다



\---



\## 2. 단계별 규칙



\### 2.1 store 선택



\* 5층\~8층 가게 목록 표시

\* 선택 결과는 participant에 귀속될 가게 정보로 사용

\* 타매장 선택도 허용



\### 2.2 category 선택



허용 값:



\* 퍼블릭

\* 셔츠

\* 하퍼



선택 즉시 기준 시간 결정:



\* 퍼블릭 = 90분

\* 셔츠 = 60분

\* 하퍼 = 60분



\### 2.3 manager 선택



\* 반드시 선택해야 한다

\* 선택한 가게 기준 담당실장 목록을 보여준다

\* 담당실장 확정 전 participant는 완전 확정 상태가 아니다



\---



\## 3. participant 확정 시점



participant는 아래 3개가 모두 선택된 순간 확정된 것으로 본다.



\* store

\* category

\* manager



이 순간부터:



\* participant 카드 표시값 갱신

\* 남은시간 계산 시작

\* 종목 집계(count) 반영



\---



\## 4. 시간 규칙



\### 4.1 시작 시점



\* 시간 시작은 participant 생성 시점이 아니라

\* `store + category + manager` 확정 완료 시점부터 시작한다



즉, `entered\_at` 기준 시점은 확정 완료 시점이다.



\### 4.2 기본 표시



\* 남은시간은 기본적으로 participant별로 표시한다



예:



\* 퍼블릭: 90 → 89 → 88분

\* 셔츠: 60 → 59 → 58분

\* 하퍼: 60 → 59 → 58분



\---



\## 5. 혼합 방 시간 표시 옵션



방에는 시간 표시 모드를 둘 수 있다.



\### 5.1 mode A: 2기준



\* 퍼블릭

\* 셔츠+하퍼



즉 기준값 2개로 본다.



\### 5.2 mode B: 3기준



\* 퍼블릭

\* 셔츠

\* 하퍼



즉 각 종목을 따로 본다.



\### 5.3 공통 원칙



\* 어떤 모드든 원본 participant 시간은 개별 participant 기준이다

\* 모드는 화면 표시/운영용 기준이다

\* 정산 source of truth를 바꾸지 않는다



\---



\## 6. participant 카드 표시 규칙



확정된 participant 카드에는 최소 아래 정보가 보여야 한다.



\* 종목

\* 담당실장

\* 가게 정보 또는 타점 표시

\* 이름 입력칸

\* 남은시간



미지정 participant는 별도 스타일로 보여야 한다.



\---



\## 7. 종목별 개수 표시



participant 목록 상단 또는 종목 라벨 영역에는

현재 방의 participant를 기준으로 종목별 개수를 표시한다.



예:



\* 퍼블릭 +1

\* 셔츠 +2

\* 하퍼 +0



규칙:



\* count는 현재 focus room의 active participant만 기준으로 계산

\* 미지정 participant는 count에 포함하지 않는다

\* category 확정 후 즉시 count 반영



\---



\## 8. 즉시 반영 규칙



가게/종목/담당실장 선택 완료 후에는 즉시 아래가 반영되어야 한다.



\* participant 카드 표시

\* 남은시간 시작

\* 종목 count 증가

\* 타점/가게 표시

\* 담당실장 표시



즉시 반영은 `fetchFocusData()` 기준 재조회 또는 동등한 상태 갱신으로 처리한다.



\---



\## 9. 금지 사항



\* 이름 입력을 종목/실장 선택보다 먼저 강제하지 않는다

\* participant 생성 직후 곧바로 entered\_at을 시작하지 않는다

\* 미지정 participant를 종목 count에 포함하지 않는다

\* 화면 표시 모드(2기준/3기준)로 원본 정산 데이터를 바꾸지 않는다



\---



\## 10. 구현 우선순위



1\. 바텀시트 단계 수정



&#x20;  \* store → category → manager

2\. participant 확정 시점 정리



&#x20;  \* entered\_at 시작 시점 고정

3\. participant 카드 표시값 보강

4\. 종목 count 표시

5\. 시간 표시 모드(2기준 / 3기준) 추가
