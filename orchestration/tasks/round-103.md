\[ROUND]

COUNTER-NAME-MATCH-FLOW-001



\[TASK TYPE]

flow redesign and controlled api/ui change



\[OBJECTIVE]

카운터에서 아가씨를 목록 선택하는 방식을 제거하고,

가게 선택 + 종목 선택 + 이름 수기 입력 기반으로

아가씨/담당실장을 자동 연결하는 흐름으로 변경한다.



\[REQUIRED BEHAVIOR]



1\. 가게 선택 후 해당 가게 아가씨 목록을 노출하지 않는다.

2\. 종목 선택 후 이름 입력칸을 바로 보여준다.

3\. 입력한 이름을 기준으로 해당 가게의 아가씨를 자동 매칭한다.

4\. 매칭 성공 시 담당실장을 자동 연결한다.

5\. 오타 또는 매칭 실패 시 후보 선택/수정 UI를 제공한다.

6\. 수정 시 수정 이력을 남긴다.



\[DETAIL RULE]



\- 예: 버닝 + 퍼블릭 + "미자" 입력

&#x20; -> 버닝 소속 후보 중 김미자 매칭

&#x20; -> 김미자의 담당실장 자동 연결



\- 예: "미지" 오입력

&#x20; -> 자동 매칭 실패 또는 불확실

&#x20; -> 후보에서 "김미자 담당실장 OOO (버닝)" 선택 가능

&#x20; -> 수정 이력 기록



\[REMOVE]

\- 가게별 출근 아가씨 목록 선택 UI 제거

\- 아가씨 직접 목록 선택 방식 제거



\[KEEP]

\- 가게 선택

\- 종목 선택

\- participant 카드 표시

\- count 표시

\- entered\_at 확정 시점 관리



\[NEW REQUIRED]

\- 이름 기반 검색/매칭

\- 후보 선택 UI

\- 수정 로그 기록



\[OUTPUT]

1\. 현재 UI에서 제거할 부분

2\. 새 입력 플로우

3\. 자동 매칭에 필요한 API/쿼리

4\. 수정 로그를 어디에 남길지

5\. 기존 participant 모델에서 재사용 가능한 필드와 부족한 필드
