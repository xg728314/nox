# PATCH — ORDER TABS + ROOM FEE RULES

## 목표
주문 탭 구조를 아래 6개로 변경한다.

- 주류
- 웨이터팁
- 룸티
- 룸티연장
- 사입
- 기타

---

## 규칙

### 1. 주류
- 기존 liquor
- 판매가 있음
- 입금가 있음

### 2. 웨이터팁
- 기존 tip
- 기존 동작 유지

### 3. 룸티
- 새 주문 타입
- 판매가 있음
- 입금가 있음
- 주류처럼 가게 매출 항목으로 처리

### 4. 룸티연장
- 새 주문 타입
- 판매가 있음
- 입금가 없음
- 전액 실장 수익으로 처리
- 가게 매출 항목 아님

### 5. 사입
- 기존 purchase
- 기존 동작 유지

### 6. 기타
- 기타 수기 입력용
- 기존 기타/예외 항목 유지 가능하면 재사용

---

## UI 수정

### 탭 라벨
탭 글자가 줄바꿈되거나 어색하게 중앙에서 깨지지 않게 수정한다.

필수:
- 한 줄 표시
- 중앙 정렬
- 버튼 높이/폭 통일
- "룸티연장"도 보기 좋게 표시
- 좁으면 가로 스크롤보다 탭 폭/폰트/패딩을 먼저 정리

권장:
- whitespace-nowrap
- min-width 조정
- flex 정렬 통일

---

## 입력 규칙

### 룸티
입력:
- 판매가
- 입금가

### 룸티연장
입력:
- 판매가만
- 입금가 입력 없음

---

## 계산 규칙

### 룸티
- store_revenue += deposit_price
- store_profit += deposit_price - cost_basis
- manager_profit += 0

### 룸티연장
- store_revenue += 0
- store_profit += 0
- manager_profit += sale_price

---

## 유지 조건
- 기존 주류 동작 깨지면 안 됨
- 기존 웨이터팁 동작 깨지면 안 됨
- 기존 사입 동작 깨지면 안 됨
- 주문 목록 누적/삭제 유지

---

## 완료조건
1. 탭이 6개로 보임
2. 글자 정렬 안 깨짐
3. 룸티 입력 가능
4. 룸티연장 입력 가능
5. 룸티는 입금가 받음
6. 룸티연장은 입금가 없음
7. TypeScript 에러 없음
